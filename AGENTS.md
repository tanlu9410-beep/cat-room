# 文档协作规则

- GitHub `docs/` 是本项目唯一正式依据。
- 不要用聊天上下文推翻已经落盘到 `docs/` 的结论。
- 想法先进 `docs/99-inbox.md`。
- 新想法先进入 `docs/99-inbox.md`，拍板后再同步到正式文档。
- 已拍板结论同步到 `docs/07-decisions.md`。
- 涉及重要更新结论时，记得同步 `docs/07-decisions.md`。
- 系统文档只写长期系统，不写一次性剧情演出。
- 流程文档只写阶段推进，不写底层实现细节。
- 不要把“内容资源”和“功能系统”混写。
- 一次性演出、长期系统、内容资源、施工切片必须分层记录，不要混写。
- 修改 `src/` 时应保持现有玩法行为不变，仅做代码质量改进。
