# 任务模板

## 这份文档是干什么的

用于创建新施工任务时的标准模板。  
复制此模板并填写具体内容。

---

## 模板

```markdown
# 任务：[任务名称]

## 目标
[一句话描述这次要做什么]

## 背景
[为什么要做这个？与哪个 FLOW 阶段或系统相关？]

## 具体要求
1. [具体要求 1]
2. [具体要求 2]
3. ...

## 不要动
- [明确列出不能碰的部分]

## 涉及文件
- [预期要修改的文件列表]

## 验收标准
- [怎么判断做完了？]
- [需要通过什么测试？]

## 审核指引
- 在 PR 中发 `@codex review`
- 检查 Gemini Code Assist 自动 review
- 确认 diff 没有超范围
```
